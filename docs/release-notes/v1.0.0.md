# mighty-diagram v1.0.0

**Turn any repository into a clean, minimalist architecture diagram — as a self-contained SVG, with nothing to install to view or render it.**

Point your AI assistant at a repo and get back a diagram of what's actually there: services, data flow, tech stack, or the primary user workflow. The first release ships the skill, Claude Code plugin packaging, a multi-platform CLI installer, an example gallery, docs, and CI.

## Highlights

- 🔎 **Repo-grounded** — reads `docker-compose`, dependency manifests, Dockerfiles, k8s/IaC, `.env`, CI, and the directory tree, then diagrams what it finds (no invented services).
- 🧭 **Three views** — **component/architecture** (default), **tech-stack**, or **user-workflow**. It proposes the best fit and lets you switch.
- 🪶 **Zero runtime dependency** — output is one `.svg` with icons inlined; renders in any browser, on GitHub, in docs, offline, and on every AI surface.
- 🎨 **Your colors** — defaults to the **PwC** brand palette; recolor everything by editing one `:root` block, or pick a preset: `pwc`, `reference`, `monochrome`, `ocean`, `forest`, `violet`.
- 📐 **Clean by construction** — hand-authored SVG in a swimlane grid: titled group columns, monochrome Lucide line icons, flow arrows, an optional deployment frame, and dashed sub-groups (each used only when the repo warrants it).

## Install

**Claude Code — plugin marketplace (one-liner):**
```
/plugin marketplace add kjchen93/mighty-diagram
/plugin install mighty-diagram@mighty-diagram
```

**CLI — any AI assistant** (Claude Code verified; also Cursor, Windsurf, Copilot, Codex, Gemini, OpenCode, Continue, Droid, KiloCode, Roo Code, Qoder, Warp, Augment, Antigravity, CodeBuddy, Kiro, Trae):
```bash
npm install -g mighty-diagram-cli
mighty-diagram init --ai claude            # current project
mighty-diagram init --ai claude --global   # all projects
mighty-diagram list                        # every supported platform
```

**Manual:**
```bash
git clone https://github.com/kjchen93/mighty-diagram
cp -r mighty-diagram/skills/mighty-diagram ~/.claude/skills/
```

## Quick start

In any repo, just ask:
```
Diagram this repository.
Show me the tech stack as a diagram.
Visualize the main user workflow.
```
The skill scans the repo, proposes a view + palette (asking once), and writes `architecture.svg`. Embed it with `![architecture](architecture.svg)`.

## What's included

- **Skill** (`skills/mighty-diagram/`): `SKILL.md`, reference guides (repo analysis, style system, SVG authoring, icons), a starter template, a worked exemplar, palette presets, and 28 bundled Lucide icons (ISC).
- **Plugin + marketplace** packaging (`.claude-plugin/`) and a `skill.json` manifest.
- **`mighty-diagram-cli`**: bundles the skill and installs it into Claude Code + 17 other assistants (`init` / `uninstall` / `update` / `list` / `versions`).
- **Example gallery** (`examples/`) from real repositories, plus contributor docs and CI (skill validation + CLI build/smoke).

## Notes & limits

- A diagram is a point-in-time snapshot — re-run to refresh after code changes.
- SVG has no auto-layout, so diagrams are bounded to ~15–25 nodes for clarity; ask for a top-level overview plus per-subsystem drill-downs on large systems.
- CLI platform paths: Claude Code is verified; other tools use the `<tool>/skills` convention — adjust in `cli/src/platforms.ts` if yours differs.
